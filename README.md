# konndate コマンド

![test](https://github.com/bannbuu/robosys2025/actions/workflows/test.yml/badge.svg)

konndateは､料理のジャンル・調理法・食材を指定すると、対応する料理名を標準出力に1行ずつ表示するコマンドです。

エラー時のメッセージは標準エラー出力に出します。

## ダウンロード・インストール

```bash

# 1. リポジトリを取得する  
   `$ git clone https://github.com/bannbuu/robosys2025.git`

# 2. ディレクトリに移動する  
   `$ cd robosys2025`

# 3. 実行権限を付ける（初回のみ）  
   `$ chmod +x konndate`    

# 4. 実行する(例:肉料理･焼く･牛)  
   `$ cat foods-肉.txt | ./konndate 肉 焼く 牛`
```
## 使い方
### コマンドの書式

`$ cat <データファイル> | ./konndate <肉|魚|麺> <調理法> <食材>`
- 第1引数: 肉 / 魚 / 麺
- 第2引数: 焼く, 煮る, 揚げる, 丼, 汁あり, 汁なし など
- 第3引数: 牛, 豚, 鳥, 白身, 赤身, 青魚, 中華, 和風, パスタ, アジア など

## 使用例

`$ cat foods-肉.txt | ./konndate 肉 焼く 牛`
   ステーキ
   焼肉  
`$ cat foods-魚.txt | ./konndate 魚 丼 赤身`
   鉄火丼
   海鮮丼  
`$ cat foods-麺.txt | ./konndate 麺 汁なし 中華`
   油そば
   汁なし担々麺

## データファイル

konndateはカレントディレクトリにある次の 3 つのファイルを読み込みます。
- `$ foods-肉.txt`
- `$ foods-魚.txt`
- `$ foods-麺.txt`

各行は次の形式で記述します。

<食材>, <調理法>:料理名

例：

牛, 焼く:ステーキ

牛, 揚げる:コロッケ

鳥, 丼:親子丼

中華, 汁あり:ラーメン

空行と、行頭が # の行は無視されます。

## 出力と終了ステータス

- 正常時
  
  条件に合う料理名を標準出力に 1 行ずつ表示します。
  
  対応する料理が 1 件以上見つかったとき、終了ステータスは 0 です。

- エラー時（メッセージは標準エラー出力に出します）

  1: 引数の数が正しくない

  2: ジャンル1 が未対応（肉 / 魚 / 麺 以外）

  3: データファイルを開けなかった

  4: データファイルは開けたが、指定した組み合わせに対応する料理がなかった

## ヘルプ

次のように -h または --help を指定すると、使い方を表示して終了します。

`$ ./konndate -h`

`$ ./konn date --help`

このときの終了ステータスは 0です｡
