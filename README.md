# konndate コマンド

![test](https://github.com/bannbuu/robosys2025/actions/workflows/test.yml/badge.svg)

konndateは､料理のジャンル・調理法・食材を指定すると、対応する料理名を標準出力に1行ずつ表示するコマンドです。
エラー時のメッセージは標準エラー出力に出します。

## 使い方
###### コマンドの書式
./konndate <ジャンル1> <ジャンル2> <ジャンル3>

<ジャンル1>: 肉 / 魚 / 麺

<ジャンル2>: 焼く, 煮る, 揚げる, 丼, 汁あり, 汁なし など

<ジャンル3>: 牛, 豚, 鳥, 白身, 赤身, 青魚, 中華, 和風, パスタ, アジア など

## 使用例
./konndate 肉 焼く 牛

./konndate 魚 丼 赤身

./konndate 麺 汁なし 中華

## 必要なソフトウェア
Python

テスト済みバージョン: 3.8, 3.9, 3.10, 3.11

## テスト環境
Ubuntu 24.04 LTS

## データファイル
konndate はカレントディレクトリにある次の 3 つのファイルを読み込みます。

foods-肉.txt

foods-魚.txt

foods-麺.txt

各行は次の形式で記述します。

<食材>,<調理法>:料理名

例：

牛,焼く:ステーキ

牛,揚げる:コロッケ

鳥,丼:親子丼

中華,汁あり:ラーメン

空行と、行頭が # の行は無視されます。

## 出力と終了ステータス
正常時

条件に合う料理名を標準出力に 1 行ずつ表示します。

対応する料理が 1 件以上見つかったとき、終了ステータスは 0 です。

エラー時（メッセージは標準エラー出力に出します）

1: 引数の数が正しくない

2: ジャンル1 が未対応（肉 / 魚 / 麺 以外）

3: データファイルを開けなかった

4: データファイルは開けたが、指定した組み合わせに対応する料理がなかった

## ヘルプ
次のように -h または --help を指定すると、使い方を表示して終了します。

./konndate -h
./konndate --help

このときの終了ステータスは 0 です。

## テスト
konndate の動作は test.bash で自動テストしています。

./test.bash

すべてのテストが通ると「OK」と表示され、終了ステータスは 0 になります。

テストでは、正常系に加えて次のような異常系も確認しています。

存在しない組み合わせ（例: 羊）で終了コード 4 になること

未対応ジャンル（例: 野菜）で終了コード 2 になること

引数不足・引数が多すぎる場合に終了コード 1 になること
